# iChat Pro 前端 GUI 设计规范

### 1. 设计目标

本系统前端 GUI 设计参考 Telegram Web 端的即时通信界面风格，采用简洁、高效、低干扰的桌面端聊天布局。界面设计以“会话优先、内容聚焦、操作路径清晰、状态反馈明确”为核心原则，使用户能够快速完成登录、查看会话、发送消息、管理联系人、修改个人资料、设置隐私与安全选项等操作。

系统整体界面以左右分栏为基础：左侧为功能导航与会话管理区域，右侧为聊天内容显示区域。左侧区域承担搜索、菜单、会话列表、设置页面等功能；右侧区域主要用于显示聊天背景、当前会话内容、消息输入框和加密状态提示。该设计能够减少页面跳转，使用户在同一主界面中完成大部分核心操作。

本系统参考 Telegram 的 GUI 设计主要包括：圆角搜索框、固定左侧会话栏、浮层式主菜单、卡片式设置页面、聊天背景纹理、亮色/暗色主题切换、简洁图标导航、消息气泡式展示等。参考对象仅用于界面结构和交互逻辑设计，不直接使用 Telegram 的商标、Logo、原始图标和专有素材。

------

### 2. 整体界面风格

系统采用现代化即时通信软件常见的轻量化视觉风格。界面整体减少复杂装饰，突出会话内容和消息交互。主要设计特征如下：

| 设计项     | 规范说明                                                     |
| ---------- | ------------------------------------------------------------ |
| 整体风格   | 简洁、现代、轻量，减少复杂边框和强装饰                       |
| 页面结构   | 左侧固定功能栏 + 右侧自适应聊天内容区                        |
| 视觉层级   | 搜索栏、会话列表、当前聊天窗口、设置面板层级分明             |
| 圆角风格   | 搜索框、菜单、按钮、消息气泡统一采用圆角                     |
| 颜色风格   | 亮色模式以白色、浅灰、蓝色为主；暗色模式以深灰、黑色、紫蓝色点缀为主 |
| 交互风格   | 鼠标悬停、点击、选中、未读提醒均有明确反馈                   |
| 空状态设计 | 未选择会话时，右侧显示主题背景或空状态提示                   |

------

### 3. 页面总体布局

#### 3.1 桌面端主界面布局

系统桌面端主界面采用左右分栏结构。

| 区域       | 建议宽度      | 主要内容                                     |
| ---------- | ------------- | -------------------------------------------- |
| 左侧功能区 | 360px ~ 420px | 菜单按钮、搜索框、会话列表、设置页面         |
| 右侧聊天区 | 自适应填充    | 聊天背景、聊天消息、消息输入区、顶部会话信息 |
| 浮层菜单   | 280px ~ 330px | 账号信息、个人资料、联系人、设置等入口       |

左侧区域固定宽度，右侧区域根据屏幕宽度自适应扩展。用户未选择具体聊天对象时，右侧区域显示默认聊天背景图案；用户选择会话后，右侧区域切换为聊天内容界面。

参考 Telegram Web 的设计，左侧栏不应占据过大面积，避免影响聊天内容展示；右侧聊天区应保留足够空间，使消息内容、图片、文件、输入框能够完整展示。

------

### 4. 信息架构设计

系统前端页面分为以下几个核心模块：

| 一级模块     | 二级功能                               | 说明                       |
| ------------ | -------------------------------------- | -------------------------- |
| 身份认证模块 | 登录、注册、退出登录                   | 负责用户身份识别           |
| 主界面模块   | 会话列表、搜索、归档会话               | 展示用户已有聊天记录       |
| 聊天模块     | 消息展示、消息发送、消息状态           | 系统核心交互区域           |
| 联系人模块   | 联系人列表、搜索联系人、添加联系人     | 管理聊天对象               |
| 个人资料模块 | 头像、昵称、账号信息                   | 展示和修改用户基本资料     |
| 设置模块     | 通用设置、通知设置、隐私安全、语言设置 | 管理系统偏好               |
| 安全模块     | 加密状态、设备会话、隐私设置           | 突出端到端加密聊天系统特征 |

### 5. 主界面设计规范

#### 5.1 顶部搜索区

左侧顶部区域由菜单按钮和搜索框组成。菜单按钮位于左上角，点击后弹出主菜单浮层；搜索框位于菜单按钮右侧，用于搜索联系人、会话和消息内容。

搜索框设计规范如下：

| 项目     | 规范                                   |
| -------- | -------------------------------------- |
| 形状     | 胶囊型圆角搜索框                       |
| 高度     | 40px ~ 44px                            |
| 背景色   | 亮色模式使用浅灰色；暗色模式使用深灰色 |
| 占位文字 | Search / 搜索                          |
| 图标     | 左侧显示放大镜图标                     |
| 交互     | 聚焦时边框或背景色轻微变化             |

搜索功能应支持模糊匹配用户昵称、账号、群聊名称和历史消息。搜索结果可直接在左侧会话列表区域中展示。

#### 5.2 会话列表区

会话列表是主界面的核心导航区域，每个会话项采用横向卡片布局，包括头像、会话名称、最后一条消息、时间、未读数量等信息。

会话项设计规范如下：

| 元素         | 说明                                             |
| ------------ | ------------------------------------------------ |
| 头像         | 圆形头像，个人会话显示用户头像，群聊显示群组头像 |
| 会话名称     | 加粗显示，位于头像右侧上方                       |
| 最后一条消息 | 使用较浅颜色显示，超出宽度省略                   |
| 时间         | 位于右侧上方，显示最近消息时间                   |
| 未读数       | 使用圆角徽标显示，颜色使用主色或强调色           |
| 选中状态     | 当前会话背景高亮                                 |
| 悬停状态     | 鼠标悬停时背景轻微变浅或变深                     |

会话列表应支持滚动浏览。大量会话存在时，滚动条应保持简洁，不影响整体视觉风格。

#### 5.3 右侧聊天背景区

参考 Telegram Web，右侧聊天区域在未选中会话时显示默认背景。背景可采用轻量纹理、渐变色或低透明度图案，使界面不显得空白，同时避免干扰聊天内容。

背景设计规范如下：

| 项目       | 规范                             |
| ---------- | -------------------------------- |
| 背景类型   | 渐变色 + 低透明度纹理图案        |
| 亮色模式   | 可使用浅绿色、浅蓝色、浅黄色渐变 |
| 暗色模式   | 可使用深黑、深紫、深蓝渐变       |
| 纹理透明度 | 控制在 5% ~ 15%，避免喧宾夺主    |
| 内容优先级 | 背景不得影响文字阅读             |

------

### 6. 主菜单设计规范

主菜单通过左上角菜单按钮打开，采用浮层卡片形式显示。菜单不占满页面，而是覆盖在左侧栏上方，体现轻量化操作。

主菜单内容建议包括：

| 菜单项     | 功能说明                         |
| ---------- | -------------------------------- |
| 用户信息区 | 显示头像、昵称、账号状态         |
| 添加账号   | 用于切换或添加其他账号           |
| 我的资料   | 进入个人资料页面                 |
| 收藏消息   | 查看用户保存的重要消息           |
| 联系人     | 进入联系人管理页面               |
| 设置       | 进入系统设置页面                 |
| 更多       | 展开更多功能，如关于系统、帮助等 |

菜单设计规范如下：

| 项目 | 规范                           |
| ---- | ------------------------------ |
| 形状 | 圆角矩形卡片                   |
| 阴影 | 使用轻微阴影区分浮层和背景     |
| 宽度 | 280px ~ 330px                  |
| 图标 | 左侧图标 + 右侧文字            |
| 行高 | 48px ~ 56px                    |
| 分组 | 账号区和功能区之间使用分隔线   |
| 动画 | 打开时可使用轻微淡入和下滑动画 |

------

### 7. 设置页面设计规范

设置页面参考 Telegram 的设置栏设计，在左侧区域内替换原有会话列表显示，右侧聊天背景区域保持不变。这样可以让用户在不跳出主界面的情况下完成设置操作。

设置页面主要结构如下：

| 区域       | 内容                                                 |
| ---------- | ---------------------------------------------------- |
| 顶部栏     | 返回按钮、标题“设置”、编辑按钮、更多按钮             |
| 用户资料区 | 大头像、昵称、在线状态                               |
| 账号信息区 | 手机号、用户名、邮箱等                               |
| 设置列表区 | 通用设置、通知、数据存储、隐私安全、语言、设备会话等 |

设置项设计规范如下：

| 元素     | 说明                             |
| -------- | -------------------------------- |
| 左侧图标 | 表示设置类别                     |
| 主标题   | 显示设置名称                     |
| 副信息   | 例如语言为 English、设备数量为 5 |
| 点击区域 | 整行可点击                       |
| 分组     | 不同类型设置之间使用浅色分隔区   |

对于端到端加密聊天软件，应重点强化“隐私与安全”设置入口，例如设备登录管理、密钥验证、黑名单管理、消息销毁时间等。

------

### 8. 聊天窗口设计规范

当用户选择某个会话后，右侧区域进入聊天窗口。聊天窗口由顶部会话栏、中部消息区和底部输入区组成。

#### 8.1 顶部会话栏

顶部会话栏用于显示当前聊天对象信息。

| 元素     | 说明                             |
| -------- | -------------------------------- |
| 头像     | 当前联系人或群聊头像             |
| 名称     | 联系人昵称或群聊名称             |
| 状态     | 在线、离线、最后上线时间         |
| 加密状态 | 显示“端到端加密已开启”等安全提示 |
| 操作按钮 | 搜索消息、更多操作、通话入口等   |

#### 8.2 消息展示区

消息展示采用左右气泡布局。自己发送的消息靠右显示，对方发送的消息靠左显示。

| 消息类型 | 展示方式             |
| -------- | -------------------- |
| 文本消息 | 气泡形式展示         |
| 图片消息 | 缩略图 + 点击预览    |
| 文件消息 | 文件卡片形式展示     |
| 系统消息 | 居中灰色小字展示     |
| 加密提示 | 居中安全提示卡片展示 |

消息气泡设计规范如下：

| 项目     | 规范                               |
| -------- | ---------------------------------- |
| 自己消息 | 靠右，使用主色浅色背景             |
| 对方消息 | 靠左，使用白色或深灰背景           |
| 圆角     | 统一圆角，靠近头像一侧可略微变化   |
| 时间     | 位于消息气泡右下角                 |
| 状态     | 已发送、已送达、已读使用小图标表示 |
| 长文本   | 自动换行，限制最大宽度             |

#### 8.3 底部消息输入区

底部输入区固定在聊天窗口底部，包括附件按钮、输入框、表情按钮和发送按钮。

| 元素     | 功能                       |
| -------- | -------------------------- |
| 附件按钮 | 上传图片、文件等           |
| 输入框   | 输入文字消息               |
| 表情按钮 | 打开表情选择面板           |
| 发送按钮 | 发送当前消息               |
| 语音按钮 | 可选功能，用于发送语音消息 |

输入框在无内容时显示占位提示“输入消息……”。当输入框为空时，发送按钮可隐藏或置灰；当用户输入内容后，发送按钮高亮显示。

### 9. 亮色与暗色主题规范

系统应支持亮色模式和暗色模式，参考截图中 Telegram Web 的主题切换效果。亮色模式适合日间使用，暗色模式适合夜间使用。

| 设计项     | 亮色模式 | 暗色模式 |
| ---------- | -------- | -------- |
| 主背景     | #FFFFFF  | #1E1E1E  |
| 左侧栏背景 | #FFFFFF  | #212121  |
| 搜索框背景 | #F1F3F5  | #2B2B2B  |
| 主要文字   | #111111  | #F1F1F1  |
| 次要文字   | #777777  | #A0A0A0  |
| 分隔线     | #EAEAEA  | #333333  |
| 主色       | #3390EC  | #8774E1  |
| 未读徽标   | #31B545  | #8774E1  |

暗色模式下应避免使用纯黑大面积背景，可以采用深灰色增强层次感。文字与背景之间需要保持足够对比度，保证可读性。

------

### 10. 组件规范

系统前端应将常用界面元素组件化，保证页面风格统一。

| 组件名称               | 用途                 | 设计要求                   |
| ---------------------- | -------------------- | -------------------------- |
| Avatar 头像组件        | 展示用户或群组头像   | 圆形，支持图片和文字缩写   |
| IconButton 图标按钮    | 菜单、返回、更多操作 | 圆形点击区域，悬停高亮     |
| SearchBar 搜索框       | 搜索联系人和消息     | 胶囊圆角，左侧搜索图标     |
| ChatListItem 会话项    | 展示会话摘要         | 支持未读数、时间、选中状态 |
| Badge 徽标             | 显示未读数量         | 圆角，颜色醒目             |
| DrawerMenu 主菜单      | 展示功能入口         | 浮层卡片，带阴影           |
| SettingsItem 设置项    | 展示设置入口         | 图标 + 标题 + 右侧信息     |
| MessageBubble 消息气泡 | 展示聊天消息         | 区分自己和对方             |
| EmptyState 空状态      | 未选择会话时展示     | 背景图案或提示文本         |
| Toast 提示框           | 操作反馈             | 短时间显示，不阻塞用户     |

------

### 11. 交互规范

系统交互设计应保证用户操作路径短、反馈明确。

| 用户操作       | 系统反馈                             |
| -------------- | ------------------------------------ |
| 点击菜单按钮   | 弹出主菜单浮层                       |
| 点击菜单外区域 | 关闭主菜单                           |
| 点击搜索框     | 搜索框进入聚焦状态                   |
| 输入搜索内容   | 实时过滤会话或联系人                 |
| 点击会话项     | 右侧切换到对应聊天窗口               |
| 发送消息       | 消息立即出现在聊天区，并显示发送状态 |
| 消息发送失败   | 显示失败图标，支持重新发送           |
| 点击设置       | 左侧栏切换为设置页面                 |
| 点击返回按钮   | 返回上一级页面                       |
| 切换主题       | 全局颜色立即切换                     |

所有关键操作都需要有反馈，例如按钮悬停、点击态、加载态、成功提示和失败提示。

------

### 12. 安全与隐私相关 GUI 设计

由于本系统定位为参考 Telegram 的安全聊天软件，界面中应体现安全通信特征。

建议加入以下设计：

| 场景           | GUI 表现                                     |
| -------------- | -------------------------------------------- |
| 当前会话已加密 | 顶部显示锁形图标和“端到端加密已开启”         |
| 密钥验证       | 在聊天详情页提供“验证安全码”入口             |
| 新设备登录     | 设置页中显示“活跃会话”数量                   |
| 账号安全       | 隐私与安全页面中提供密码、设备、黑名单等设置 |
| 消息销毁       | 聊天设置中提供“定时销毁消息”选项             |
| 敏感操作       | 删除聊天、退出账号等操作弹出确认框           |

安全提示不应过度打扰用户，但必须在关键位置明确展示。例如聊天窗口顶部、隐私设置页、设备管理页等。

------

### 13. 响应式设计规范

虽然系统主要面向 Web 端或桌面端，但仍应考虑不同屏幕尺寸。

| 屏幕宽度       | 布局策略                               |
| -------------- | -------------------------------------- |
| 大于 1200px    | 左侧栏 + 右侧聊天区完整显示            |
| 768px ~ 1200px | 左侧栏保持固定，右侧自适应             |
| 小于 768px     | 左侧列表和聊天窗口采用单页切换         |
| 小于 480px     | 隐藏部分文字，仅保留必要图标和核心操作 |

移动端不建议强行保留左右分栏，而应采用“会话列表页 → 聊天详情页”的页面跳转模式。

### 14. 前端实现建议

本系统前端可采用 HTML5、CSS3、JavaScript 与 Tailwind CSS 实现界面样式，使用 WebSocket 实现实时消息刷新。若采用 Django 技术栈，可结合 Django 模板负责基础页面渲染，使用 HTMX 或原生 JavaScript 完成局部刷新，使用 Channels 处理实时通信。

建议前端目录结构如下：

```text
frontend/
├── pages/
│   ├── login.html
│   ├── register.html
│   ├── chat.html
│   └── settings.html
├── components/
│   ├── sidebar.html
│   ├── chat_list_item.html
│   ├── message_bubble.html
│   ├── drawer_menu.html
│   └── settings_item.html
├── static/
│   ├── css/
│   │   ├── base.css
│   │   ├── theme.css
│   │   └── chat.css
│   ├── js/
│   │   ├── chat.js
│   │   ├── websocket.js
│   │   └── theme.js
│   └── images/
│       └── chat-bg.svg
```

CSS 变量建议如下：

### 16. GUI 设计验收标准

为了保证最终实现效果与设计规范一致，前端界面应满足以下验收标准：

| 验收项     | 标准                                               |
| ---------- | -------------------------------------------------- |
| 页面完整性 | 登录、注册、主界面、聊天页、设置页均可正常访问     |
| 布局一致性 | 主界面保持左侧栏 + 右侧聊天区结构                  |
| 视觉一致性 | 搜索框、按钮、菜单、设置项风格统一                 |
| 主题适配   | 支持亮色模式和暗色模式                             |
| 交互反馈   | 点击、悬停、选中、加载、失败状态均有反馈           |
| 消息展示   | 能区分自己消息和对方消息                           |
| 安全提示   | 聊天界面能显示加密状态或隐私提示                   |
| 响应式适配 | 不同屏幕尺寸下界面不发生严重错位                   |
| 文档一致性 | 页面功能应与用例图、时序图、类图、系统实现截图一致 |

### 17. 小结

本系统 GUI 设计以 Telegram Web 端为主要参考对象，采用左侧会话导航与右侧聊天内容区相结合的布局方式。该设计能够突出即时通信系统的核心功能，使用户在进入系统后可以快速搜索联系人、查看会话、发送消息并管理个人设置。通过亮色/暗色主题、浮层式菜单、卡片式设置页、消息气泡、未读徽标和加密状态提示等设计，系统在视觉风格、交互体验和安全感知方面均保持较好的统一性。
