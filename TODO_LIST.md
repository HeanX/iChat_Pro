# iChat Pro TODO List

本文档用于确定一期开发范围。团队确认后，将每个任务创建为 GitHub Issue，由负责人认领并从独立分支提交 Pull Request。

## 0. 协作约定

- [ ] 在 GitHub 为 `main` 开启分支保护，禁止直接 push
- [ ] 合并 PR 前至少需要 1 位协作者 Review
- [ ] 每个任务对应 1 个 Issue、1 个分支、1 个 PR
- [ ] PR 描述中使用 `Closes #<issue-number>` 关联 Issue
- [ ] 功能分支命名为 `feature/<issue-number>-<short-name>`
- [ ] 修复分支命名为 `fix/<issue-number>-<short-name>`
- [ ] 文档分支命名为 `docs/<issue-number>-<short-name>`
- [ ] 合并方式统一使用 Squash and merge
- [ ] 不提交 `.venv/`、`.idea/`、`db.sqlite3`、本地密钥或 `.env`

- [ ] 派生会话密钥

**验收标准：**

- 网络请求、WebSocket 数据和数据库中均不出现私聊明文

**目标：** 为每位有效群成员生成独立密文副本。

- [ ] 获取当前有效群成员和公钥
